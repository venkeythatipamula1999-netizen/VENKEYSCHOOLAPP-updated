Security Audit

✅ In Place

  • JWT authentication on all protected routes
  • Firebase Auth (Email/Password) — no fallback
  • Firestore security rules deployed (authenticated access only)
  • sync_errors & scan_rejection_logs blocked from client
  • schools collection blocked from client
  • promotionHistory read-only from client
  • CORS restricted to APP_URL + Vercel admin domain
  • Role-based access enforced at API level
  • Teacher send-doc permission check (own classes only)
  • Firestore batch writes (max 200/batch — safe)
  • PIN system fully removed (no weak auth)
  • All env vars set via secrets (not hardcoded)
  • Request body size limited to 10mb
  • Auto clock-out scheduled at 7 PM daily

⚠️ Recommendations

  • Firestore write rules still allow all authenticated users — tighten per role when ready
  • parent_accounts rule references resource.data.uid — ensure this field is set on creation
  • Rate limiting not yet added to auth endpoints
  • No refresh token rotation — JWT expiry is single-use
  • File upload (Send to Parents) uses URL — ensure Firebase Storage rules match
  • SUPER_ADMIN_KEY should be rotated periodically
  • Logs should be scrubbed of PII before production
